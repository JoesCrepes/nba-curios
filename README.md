# NBA Curios

A collection of NBA data visualizations and analysis projects.

## Projects

### 1. [Two-Point % Explorer](./projects/nba-two-point-percentage)
Find the highest single-season two-point field goal percentage in NBA history. Search across the modern era (1996–97 to present) using natural language queries, filtered by position and season. No LLM required — query parsing is regex-based and runs entirely in the browser.

**Ask things like:**
- "highest two-point percentage center ever"
- "best free throw shooter guard 2023-24"
- "most points per game forward"

### 2. [Season Trajectories](./projects/nba-season-trajectories)
Interactive visualization comparing win trajectories across notable NBA seasons. Features forecasting for in-progress seasons.

**Featured seasons:**
- 2015-16 Golden State Warriors (73-9)
- 1995-96 Chicago Bulls (72-10)
- 2012-13 Oklahoma City Thunder
- 2024-25 Oklahoma City Thunder (current)

## Data Pipeline

Player season stats are pre-fetched from the NBA API and committed as static JSON files. The web app filters and ranks entirely client-side — no backend server required.

### First-time setup (seed historical data)

Run this once locally to populate all seasons from 1996–97 to present:

```bash
cd projects/nba-two-point-percentage/scripts
pip install -r requirements.txt
python fetch_data.py          # ~20 min, fetches all seasons
```

Commit the generated `public/data/player_seasons.json` and push. After that, data stays fresh automatically.

### Ongoing updates

Two mechanisms keep data current:

1. **Vercel build** — `--update-current` runs on every deploy, refreshing the current season in seconds.
2. **GitHub Actions** — `update-player-data.yml` runs daily at 9 AM ET, commits updated data, and triggers an automatic Vercel redeploy.

## Tech Stack

- **Data**: Python 3.11+, nba_api, pandas
- **Frontend**: React 19, TypeScript, Vite, React Router
- **Deployment**: Vercel (static, no backend)
